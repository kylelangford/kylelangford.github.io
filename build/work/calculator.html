Form Validation and PHP Development

Savings Calculator

Ask Enel X wanted to develop a savings calculator as a lead generation tool. The form would feature a postal code field that mapped to the Demand Response programs available for those locations(US,CA), as well as the $/kW-year in Demand Response earnings. The user would enter the industry average kW by choosing their industry, which would auto-populate the curtailable kW amount, with an option to input their own curtailable kW value if they have an estimate.

Contribution Good forms help users recognize, diagnose, and recover from errors. A lot of time was spent creating real time validation for the postal code as well other input fields. Using a combination of HTML5 Validation and Javascript. We wanted this form to be engaging so time was spent on the look and feel of the slide input.

How do we test this?

Debugging was another important consideration because there were over 400k postal values. I designed and built a debug option that would display the dynamic data for easier quality assurance

Additional Detail

The form is hosted on a seperate server and added to the page with an iframe. The reasons for this a not very interesting. But, I was able to make the iframe responsive using the post_message api. This works by sending height information from the client to the parent sit with an iframe. Cool!
